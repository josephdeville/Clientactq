# PQS Message Template

## Pain-Qualified Segment (PQS) Message Structure

### The Framework: Mirror-Insight-Ask

The goal is to make prospects think: **"How did they know we're dealing with exactly this?"**

---

## Template Structure

### Subject Line
```
[3-5 words, reference specific data point or deadline]
```

Examples:
- `FedRAMP deadline by Q4`
- `Scaling ${function} after Series ${round}`
- `${legacy_tech} migration timeline`

---

### Sentence 1: MIRROR
**Purpose:** State verifiable data point reflecting their specific situation.

**Format:**
```
Your [observable data point 1] [combined with/alongside] [observable data point 2]
suggests you're [logical inference about their situation].
```

**Example:**
```
Your FedRAMP Marketplace listing combined with your recent ISSM job posting
suggests you're building compliance capability under deadline pressure.
```

---

### Sentences 2-3: INSIGHT/LEARNING
**Purpose:** Share non-obvious pattern from peer companies.

**Format:**
```
Most [similar companies in situation] initially try [common first approach]—
but the ones who [achieved desired outcome] discovered that [specific tactic/approach]
[quantified result]. They avoided [common costly mistake].
```

**Example:**
```
Most GovCon companies in this position initially try hiring an internal compliance
team—but the ones who achieved ATO on schedule discovered that pairing fractional
expertise with your existing team reduced timeline by 40%. They avoided the
documentation death spiral that derails 60% of first-time submissions.
```

---

### Sentence 4: CURIOSITY ASK
**Purpose:** Low-friction question inviting dialogue.

**Format:**
```
Curious if you're seeing [specific symptom of the problem]?
```

**Example:**
```
Curious if documentation gaps are showing up for your team?
```

---

## Complete Example

```
Subject: FedRAMP deadline by Q4

Your FedRAMP Marketplace listing combined with your recent ISSM job posting
suggests you're building compliance capability under deadline pressure.

Most GovCon companies in this position initially try hiring an internal
compliance team—but the ones who achieved ATO on schedule discovered that
pairing fractional expertise with your existing team reduced timeline by 40%.
They avoided the documentation death spiral that derails 60% of first-time
submissions.

Curious if documentation gaps are showing up for your team?
```

## Variables

| Variable | Description | Source |
|----------|-------------|--------|
| `${observable_data_point_1}` | First verifiable fact | Tier 1/2 data |
| `${observable_data_point_2}` | Second verifiable fact | Tier 1/2 data |
| `${logical_inference}` | What the data implies | Analysis |
| `${peer_companies}` | Similar companies descriptor | ICP |
| `${common_approach}` | What most try first | Market research |
| `${desired_outcome}` | Success definition | Value prop |
| `${specific_tactic}` | Differentiated approach | Solution |
| `${quantified_result}` | Measurable improvement | Case studies |
| `${costly_mistake}` | What to avoid | Pain points |
| `${problem_symptom}` | Observable challenge | Research |

---

## Validation Checklist

Before sending, verify:

- [ ] Both data points are verifiable through public sources
- [ ] Inference follows logically from the data
- [ ] Peer insight is non-obvious (not common knowledge)
- [ ] Quantified result is credible and documented
- [ ] Ask is low-friction (not asking for a meeting)
- [ ] Subject line is 3-5 words
- [ ] Total email is 4-6 sentences
